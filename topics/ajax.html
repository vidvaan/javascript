What is AJAX?

AJAX (Asynchronous JavaScript and XML) is a technique used to create fast and dynamic web pages. It allows web applications to send and receive data from the server asynchronously, without reloading the entire page. AJAX is widely used to improve user experience by updating parts of a web page dynamically.

Key Features of AJAX:

  • Asynchronous communication with the server.
  • Uses XMLHttpRequest or Fetch API to send and receive data.
  • Supports data formats like JSON, XML, HTML, and plain text.

What is a REST API?

A REST (Representational State Transfer) API is a web service that follows REST architectural principles. It allows clients to interact with a server through a standard set of HTTP methods like GET, POST, PUT, DELETE, etc.

Key Features of REST API:

  • Stateless communication: Each request is independent and contains all the necessary information.
  • Supports multiple data formats (e.g., JSON, XML).
  • Uses standard HTTP methods (GET, POST, PUT, DELETE, etc.).

What is Postman?

Postman is a popular tool for testing APIs. It provides a user-friendly interface for sending requests to an API and analyzing the responses. Postman supports all HTTP methods and allows for creating collections, environment variables, and automated tests.

Steps to Test APIs with Postman:

  1. Install Postman from here.
  2. Open Postman and create a new request.
  3. Select the HTTP method (e.g., GET, POST, PUT, DELETE).
  4. Enter the API endpoint (e.g., https://jsonplaceholder.typicode.com/posts).
  5. Add headers, query parameters, or request body if needed.
  6. Click the "Send" button to execute the request.
  7. Analyze the response in the Postman interface.

1. GET Request

This example demonstrates a GET request to fetch data from a server.


fetch('https://jsonplaceholder.typicode.com/posts/1')
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.error('Error:', error));

2. POST Request

This example demonstrates a POST request to send data to a server.


fetch('https://jsonplaceholder.typicode.com/posts', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({
        title: 'foo',
        body: 'bar',
        userId: 1
    })
})
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.error('Error:', error));

3. PUT Request

This example demonstrates a PUT request to update data on a server.


fetch('https://jsonplaceholder.typicode.com/posts/1', {
    method: 'PUT',
    headers: {
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({
        id: 1,
        title: 'Updated Title',
        body: 'Updated body content',
        userId: 1
    })
})
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.error('Error:', error));

4. DELETE Request

This example demonstrates a DELETE request to remove data from a server.


fetch('https://jsonplaceholder.typicode.com/posts/1', {
    method: 'DELETE'
})
    .then(response => response.json())
    .then(data => console.log(data))
    .catch(error => console.error('Error:', error));

5. OPTIONS Request

This example demonstrates an OPTIONS request to check available HTTP methods.


fetch('https://jsonplaceholder.typicode.com/posts', {
    method: 'OPTIONS'
})
    .then(response => console.log(response.headers))
    .catch(error => console.error('Error:', error));

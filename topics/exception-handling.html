Exception Handling in JavaScript

Exception handling in JavaScript allows developers to handle runtime errors gracefully and maintain the flow of the program. JavaScript provides the try, catch, finally, and throw statements for error handling.

1. The try and catch Statements

The try block contains code that may throw an error, while the catch block handles the error if it occurs.


try {
    let result = 10 / 0;
    console.log(result); // Outputs: Infinity, no error here
    let invalid = JSON.parse("invalid JSON"); // This will throw an error
} catch (error) {
    console.log("An error occurred:", error.message); // Outputs the error message
}

2. The finally Block

The finally block contains code that will always execute, regardless of whether an error occurs or not.


try {
    console.log("Trying...");
    throw new Error("Something went wrong");
} catch (error) {
    console.log("Caught an error:", error.message);
} finally {
    console.log("This will always run.");
}

3. Throwing Custom Errors

You can throw custom errors using the throw statement. This is useful for handling specific conditions in your code.


function divide(a, b) {
    if (b === 0) {
        throw new Error("Division by zero is not allowed.");
    }
    return a / b;
}

try {
    console.log(divide(10, 0));
} catch (error) {
    console.log("Custom error:", error.message);
}

4. Nested try-catch Blocks

You can nest try-catch blocks to handle different levels of exceptions.


try {
    try {
        let data = JSON.parse("invalid JSON");
    } catch (innerError) {
        console.log("Inner error caught:", innerError.message);
        throw new Error("Outer error due to inner error");
    }
} catch (outerError) {
    console.log("Outer error caught:", outerError.message);
}

5. Using try-catch with Promises

While working with promises, you can use try-catch inside async functions to handle errors.


async function fetchData() {
    try {
        let response = await fetch("https://api.example.com/data");
        if (!response.ok) {
            throw new Error("Network response was not ok");
        }
        let data = await response.json();
        console.log(data);
    } catch (error) {
        console.log("Error fetching data:", error.message);
    }
}

fetchData();

Summary

JavaScript's exception handling mechanism helps improve the robustness of your code by handling runtime errors gracefully. Use try-catch for synchronous errors, finally for cleanup code, and throw for custom error messages.
